02·Definition

Context engineering, not prompt engineering.

A context package is the complete informational environment an AI agent needs to operate: the system prompt, the data schemas it expects, the tools it can call via MCP, the memory patterns it uses, and the guardrails enforcing safe decisions. A context package for BOL processing includes all of these — not just a prompt.

Two gates run on every call. A pre-LLM input check validates schema, tenant, and rate. A post-LLM action check evaluates the proposed decision against guardrails and routes for human approval where required. Nothing executes without passing both.

09·FAQ

Engineering questions, answered concretely.

Q1What's the difference between context engineering and prompt engineering?+

Prompt engineering produces a single instruction. Context engineering produces the entire informational environment an agent operates inside — system prompt, schemas, tools, memory, guardrails, audit. We ship that whole package per workflow, versioned and tested, so your team writes integration code, not retrieval glue.

Q2Why MCP and not LangChain?+

MCP is a stable, model-side protocol with clear tool contracts and a healthy SDK. LangChain is a framework that wraps it. We are infrastructure for other engineers; we don't ship abstractions our customers have to learn to debug. The Anthropic MCP SDK is the entire dependency.

Q3How does multi-tenancy work?+

Supabase Row-Level Security, scoped per tenant id, enforced at the database. Memory, audit logs, custom guardrail rules and approved-carrier lists are isolated. JWTs flow from Clerk; rules cannot read across tenants even with a compromised application key.

Q4Can I write custom guardrail rules?+

Yes. Growth supports custom cost thresholds; Scale and Enterprise support full custom rule sets, including pre-LLM and post-LLM gates, escalation roles, and approver matrices. Rules are declarative and live alongside our standard set.

Q5Do you offer a free tier?+

No. The Starter tier at $149/mo is the on-ramp — it gives you the BOL and Carrier Rates modules, 1,000 calls, and 30 days of audit retention. The cost of running this responsibly is real, and a free tier would either capture the wrong users or compromise the architecture.
